## 拉取代码运行

使用参考 [使用 npm 运行](#使用-npm-运行)，该方法除了不用安装 Node 依赖外，其它步骤都相同。

不安装依赖那么代码哪里来呢？（想起某人问我，你代码就一行，几十个字符，怎么可能运行起来。npm：？？？）

- npm code：https://www.npmjs.com/package/@asunajs/dist?activeTab=code
- 指定版本：https://www.npmjs.com/package/@asunajs/dist/v/0.0.1?activeTab=code
- npm cpode - 淘宝镜像：https://npmmirror.com/package/@asunajs/dist/files?version
- 指定版本：https://npmmirror.com/package/@asunajs/dist/files/index.d.ts?version=0.0.1

值得注意的是，淘宝镜像可以之间通过名称访问文件，这方便使用青龙直接拉取代码。例如访问 `caiyun.cjs.js` 文件

https://registry.npmmirror.com/@asunajs/dist/0.0.1/files/caiyun.cjs.js

将 `0.0.1` 替换为具体版本号即可，也可以替换为 `latest` 获取最新版本。

## 使用 npm 运行

### 安装 Node 依赖

在侧边栏依赖管理中添加 Node 依赖

```bash frame="none"
@asunajs/xxxxx

# 例如
@asunajs/caiyun
```

:::tip[你知道吗？]
如果出现类似于

```bash frame="none"
开始安装依赖 bilioutils，开始时间 2023/8/15 20:30:31
WARN using --force Recommended protections disabled.
```

依赖一直（长时间）在安装中可能是仓库源地址的问题，切换 npm 源。

:::

### 创建脚本

`asign.js` 或其它任何合法文件名，后缀为 `.js`，内容如下（xxxx 替换为具体模块名）：

```js
require('@asunajs/xxxx').run();
```

上面代码将使用默认配置路径，想要自定义请向 run 函数传递路径参数 `run('xxxxx')`;

### 更新脚本

删除依赖 `@asunajs/xxxx`，并重新安装该依赖即可（并不一定有效）。

自动更新可以手动添加一个任务，运行命令为 `pnpm remove @asunajs/xxxx && pnpm add @asunajs/xxxx`（并不一定有效，与版本有关）
